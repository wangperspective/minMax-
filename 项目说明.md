# 后台管理系统

一套完整的前后端分离后台管理系统，可直接运行、编辑、联调。

## 技术栈

### 后端
- Node.js + Express
- JWT 身份验证
- MySQL 数据库
- RESTful API

### 前端
- Vue3 + 原生 JavaScript
- Vue Router 路由
- Ant Design Vue UI 组件库
- ECharts 图表
- Pinia 状态管理

## 功能模块

1. **登录/退出** - JWT 令牌认证，自动刷新
2. **用户管理** - 增删改查、状态控制、密码重置
3. **角色管理** - 角色配置、权限分配
4. **菜单管理** - 树形菜单、权限配置
5. **数据大屏** - ECharts 可视化图表

## 快速启动

### 1. 数据库准备

```bash
# 进入后端目录
cd 后端

# 执行数据库初始化脚本
mysql -u root -p < database.sql
```

### 2. 启动后端

```bash
cd 后端

# 安装依赖
npm install

# 复制环境变量文件
cp .env.example .env

# 编辑 .env 文件，配置数据库密码
# DB_PASSWORD=your_mysql_password

# 启动服务
npm run dev
```

后端服务运行在 http://localhost:3001

### 3. 启动前端

```bash
cd 前端

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端服务运行在 http://localhost:5173

## API 接口

### 认证
- `POST /api/auth/login` - 登录
- `POST /api/auth/refresh` - 刷新令牌
- `GET /api/auth/current` - 获取当前用户
- `POST /api/auth/logout` - 退出

### 用户
- `GET /api/users` - 用户列表
- `POST /api/users` - 创建用户
- `PUT /api/users/:id` - 更新用户
- `DELETE /api/users/:id` - 删除用户

### 角色
- `GET /api/roles` - 角色列表
- `POST /api/roles` - 创建角色
- `PUT /api/roles/:id` - 更新角色

### 菜单
- `GET /api/menus/tree` - 菜单树
- `POST /api/menus` - 创建菜单
- `PUT /api/menus/:id` - 更新菜单

### 统计
- `GET /api/statistics/dashboard` - 首页统计

## 常见问题

### 数据库连接失败
1. 检查 MySQL 服务是否启动
2. 检查 `.env` 文件中的数据库配置
3. 确保数据库已创建（执行 database.sql）

### 前端无法连接后端
1. 检查后端服务是否启动（访问 http://localhost:3001/health）
2. 检查前端 `.env.development` 中的 API 地址

### 登录失败
1. 确认数据库已执行初始化脚本
2. 使用默认账号：admin / admin123

## 生产部署

### 后端
1. 修改 `.env` 配置
2. 设置 `NODE_ENV=production`
3. 使用 PM2 管理进程

### 前端
1. 执行 `npm run build` 构建
2. 将 `dist` 目录部署到 Web 服务器
3. 配置 Nginx 反向代理
